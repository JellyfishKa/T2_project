# AGILE LITE КРАТКИЙ СТАРТ

**Облегченная Agile методология для быстро движущихся команд**

## Основные концепции

### 1. Kanban Board (Без спринтов)
```
BACKLOG → TO DO → IN PROGRESS → REVIEW → DONE

- BACKLOG: Будущие задачи (не начиналась)
- TO DO: Выбрана на эту неделю
- IN PROGRESS: Сейчас работаю
- REVIEW: Готово, ждёт проверки
- DONE: Завершено & merged
```

### 2. Еженедельное планирование (Не спринты)
- **Когда**: Понедельник 10:00
- **Продолжительность**: 30 минут
- **Кто**: Вся команда
- **Что**: Выбрать 12-15 задач на неделю из BACKLOG

### 3. Ежедневная работа (Асинхронно)
- **Когда**: Анytime during day
- **Что**: Переместить свои задачи через board
- **Без ежедневных стендапов**: Обновляй Jira
- **Блокирован?**: Сразу постой в #blockers Slack

### 4. Пятничное демо + Ретро
- **Когда**: Пятница 16:00
- **Продолжительность**: 1 час
- **Демо (30 мин)**: Показать что завершили
- **Ретро (30 мин)**: Обратная связь команды + 1 улучшение

## Еженедельный ритм

### Понедельник (Начало недели)

**09:50 AM** - Подготовка PM
```
1. Открыть Jira доску
2. Подготовить 15 задач для выбора команды
```

**10:00 AM** - Планирование (30 мин)
```
1. Команда смотрит BACKLOG
2. Backend выбирает 5-6 задач
3. Frontend выбирает 3-4 задачи
4. ML выбирает 2-3 задачи
5. PM выбирает 2-3 задачи (PM-1 to PM-3, QA-1, etc)
6. Переместить выбранные в TO DO колонку
7. Назначить себе
8. Готово! Начинаем работать.
```

**После планирования**:
- Начать работать на своих задачах
- Переместить в IN PROGRESS когда начинаешь
- Постить блокеры в #blockers если застрял

---

### Вторник-Четверг (Рабочие дни)

**Утро**:
- [ ] Проверить Jira доску за своими задачами
- [ ] Выбрать одну задачу для работы
- [ ] Переместить в IN PROGRESS
- [ ] Работать!

**Когда завершил**:
- [ ] Переместить в REVIEW колонку
- [ ] Отметить рецензента в #code-review Slack
- [ ] Выбрать следующую задачу

**Когда одобрено**:
- [ ] Смёржить в main ветку
- [ ] Переместить в DONE колонку
- [ ] Выбрать следующую задачу

---

### Пятница (Ретроспектива)

**15:50 PM** - Подготовка к демо (10 мин)
```
1. Открыть Jira доску
2. Посчитать задачи в DONE
3. Подготовить talking points
```

**16:00 PM** - Демо + Ретро (1 час)

**Часть Демо (30 мин)**:
```
1. Открыть Jira доску
2. "Вот что мы завершили на этой неделе"
3. Каждый показывает свои DONE задачи
4. Краткое объяснение каждой
5. Q&A
```

**Часть Ретро (30 мин)**:
```
1. "Что прошло хорошо?" → команда делится
2. "Что было сложно?" → команда делится
3. "Одно что улучшить?" → голосуем & выбираем одно
4. "На следующей неделе фокусируемся на: [улучшение]"
5. Отмечаем победы!
```

## Состояния задач (Объяснение)

### BACKLOG
- Не начиналась
- Не назначена никому
- Готова когда команда выберет

### TO DO
- Выбрана на эту неделю (Понедельник планирование)
- Назначена кому-то
- Готова к началу

### IN PROGRESS
- Сейчас работаю
- Ты работаешь прямо сейчас
- Начал кодить/дизайнить

### REVIEW
- Ты думаешь что готово
- Ждёт проверки кода
- Рецензент: проверить качество + корректность
- Временная рамка: <4 часов

### DONE
- Код проверен
- Тесты пройдены
- Смёржено в main
- Готово к использованию!

## Если я застрял (заблокирован)?

**Сразу** (не ждать):
```
1. Telegram/Jira: "@PM заблокирован на [task]: [причина]"
2. PM ответит в течение 30 мин
3. Разблокирован или получил обходной путь
4. Продолжаю работу!
```

**Никогда**:
- Не жди молча
- Не сиди застрявший часами
- Не начинай другое без уведомления PM

## Метрики которые мы отслеживаем

### Еженедельно
- **Velocity**: Задач завершено за неделю (целевой: 12+)
- **Blockers**: Среднее блокеров в день (целевой: <1)
- **Code review**: Время ответа (целевой: <4 часов)

### Ежемесячно
- **Team morale**: Шкала 1-10 (целевой: 8+)
- **Completion**: Задач завершено vs запланировано (целевой: 90%+)
- **Quality**: Ошибок найдено при проверке (целевой: <1 на задачу)
