## Роли в команде

### Backend разработчик
**Ответственность**: API, интеграция с LLM, база данных

**Задачи на эту неделю (Неделя 1, пример)**:
- BE-1: Настройка проекта (GitHub, requirements.txt, venv)
- BE-2: Получить доступ к LLM (GigaChat + HuggingFace токены)
- BE-3: BaseLLMClient (абстрактный класс)
- BE-4: GigaChatClient реализация
- BE-5: CotypeClient реализация
- BE-6: T-ProClient реализация (stretch задача)

**Ежедневный workflow**:
1. Утро: Проверить Jira за своими задачами
2. Выбрать одну задачу (или продолжить вчерашнюю)
3. Переместить в IN PROGRESS
4. Кодить, тестировать, коммитить
5. Когда готово: Переместить в REVIEW
6. Отметить рецензента в #code-review
7. Когда одобрено: Смёржить & переместить в DONE

**Стандарты кода**:
- Покрытие тестами: 80%+
- Нет TODO комментариев (финиш перед коммитом)
- Следовать PEP 8 стилю
- Коммит сообщения: `BE-X: Краткое описание`

**Блокеры**:
- Не можешь получить токен? → Постой в #blockers сразу
- Нужна спецификация frontend? → Посмотри API_CONTRACT.md в docs
- Тесты падают? → Спаррируй с кем-то, не жди

### Frontend разработчик
**Ответственность**: UI, компоненты, пользовательский опыт

**Задачи на эту неделю (Неделя 1, пример)**:
- FE-1: Инициализация Vue проекта с Vite
- FE-2: Создание layout & навигации
- FE-3: Построение API сервиса (mock данные)
- FE-4: Страница Dashboard
- FE-5: Форма оптимизации

**Ежедневный workflow**:
1. Проверить Jira за назначенными задачами
2. Начать с дизайна/структуры компонентов
3. Использовать mock данные (backend предоставит реальные в Неделе 2)
4. Тестировать локально
5. Запушить PR с описанием
6. Отметить рецензента в #code-review

**Стандарты кода**:
- TypeScript strict mode
- Документация компонентов (комментарии)
- Responsive дизайн (мобильный-first)
- Доступность (навигация с клавиатуры)
- Коммит сообщения: `FE-X: Краткое описание`

**Ждёшь API?**:
- Используй API_CONTRACT.md для видения ожидаемой структуры
- Создай mock interceptor в axios
- Легко поменять на реальные данные позже

## Каналы коммуникации

### telegram

**#blockers** (Только чрезвычайные)
- Используй когда: Застрял и не может продолжить
- Время ответа: PM в течение 30 мин
- Пример: "@PM заблокирован на BE-2: не могу получить GigaChat токен"

**#code-review** (Проверка PR)
- Используй когда: Код готов к проверке
- Время ответа: Рецензент <4 часа
- Пример: "PR #123 готов к проверке @backend-dev"

**#general** (Объявления)
- Используй для: Важные обновления, победы команды, объявления
- Пример: "Неделя 1 завершена!"

### Jira

**Основная коммуникация**: Используй комментарии в Jira задачах
- Ссылай код/доки в комментариях
- Объясняй решения
- Отмечай людей когда нужно

**Пример**:
```
BE-2 Комментарий: "Используем Cotype как fallback пока жду GigaChat токен (expected EOD)"
```

## Сотрудничество

### Процесс проверки кода

**Разработчик**:
1. Запуши код на GitHub
2. Создай Pull Request с описанием
3. Отметь рецензента в #code-review Slack
4. Пример: "PR #123 готов к проверке, реализует BE-4"

**Рецензент**:
1. Проверить код в течение 4 часов
2. Попросить изменения если нужно ИЛИ одобрить
3. Рецензент одобряет → Разработчик мёржит
4. Разработчик переместить задачу в DONE в Jira

**Чек-лист рецензента**:
- [ ] Код следует style guide
- [ ] Тесты проходят (80%+ покрытие)
- [ ] Нет очевидных ошибок
- [ ] Документация/комментарии ясны
- [ ] Коммиты чистые

**Одобри если**:
- Решает задачу полностью
- Качество кода хорошее
- Тесты проходят

**Запроси изменения если**:
- Нет тестов
- Код сложен для чтения
- Проблемы производительности
- Задача не завершена

### Обмен знаниями

**Когда учишь что-то новое**:
1. Документируй в комментариях PR
2. Ссылай на внешние ресурсы
3. Постой в Slack если полезно для команды

**Пример**: "Использовал environment variables для API ключей (docs: https://12factor.net/config)"

## Решение проблем

### Если заблокирован (на задаче, не может продолжить)

**Шаг 1**: Попробуй решить (15 мин)
```
- Гугли ошибку
- Посмотри документацию
- Спроси товарища в Slack (5 мин)
```

**Шаг 2**: Escalate (постой в #blockers)
```
"@PM заблокирован на BE-2: GigaChat API возвращает 403 ошибку"
```

**Шаг 3**: PM разблокирует
```
- Предлагает обходной путь (Cotype fallback)
- Исследует решение
- Предоставляет timeline
```

**Шаг 4**: Продолжи
```
- Используй обходной путь если доступен
- Или жди решение
- Не сиди без дел!
```

---

### Если конфликт

**Между тобой и товарищем**:
1. Поговори прямо в Slack
2. Если не решилось в 30 мин: Escalate PM
3. PM faciliate решение в 1-on-1 звонках

**Несогласие с процессом**:
1. Обсуди в ретро (Пятница)
2. Команда голосует за подход
3. Реализуй что выиграло
